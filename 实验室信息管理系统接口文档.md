# 实验室信息管理系统（LIMS）接口文档

本系统基于原有 AgileBoot 系统二次开发，新增“实验室信息管理”能力，包含两大模块：
- 用户管理（实验室用户，独立于系统用户 SysUser）
- 成果管理（成果、作者、我的成果、字典）

本文档梳理以上模块主要功能与接口，便于前后端协作与联调。

## 通用约定
- 认证：JWT，除开放接口外均需携带 Authorization: Bearer <token>
- 返回结构：统一使用 ResponseDTO
  - 成功：{ code: 0, msg: "success", data: any }
  - 失败：{ code: 非0, msg: 错误信息 }
- 分页：PageDTO，常用字段 pageNum, pageSize, total, rows
- 文档与测试：Swagger UI /swagger-ui.html，选择“实验室管理API”分组

## 模块一：用户管理

说明：实验室维护一套独立的用户体系（lab_user）。本模块提供个人信息查询、管理员 CRUD、密码与状态等操作。

### 1.1 实验室用户 API（LabUserController）
基路径：/lab/users

- GET /profile
  - 用途：获取当前登录用户的个人信息
  - 权限：已登录
  - 返回：LabUserProfileDTO

- GET /{userId}
  - 用途：获取指定用户的个人信息
  - 权限：lab:user:query 或 本人可见
  - 返回：LabUserProfileDTO

- GET /username/{username}
  - 用途：按用户名获取个人信息
  - 权限：lab:user:query 或 本人可见
  - 返回：LabUserProfileDTO

- GET /{userId}/exists
  - 用途：检查用户是否存在
  - 权限：已登录
  - 返回：Boolean

- GET /username/{username}/exists
  - 用途：检查用户名是否存在
  - 权限：已登录
  - 返回：Boolean

- GET /test/health
  - 用途：健康检查
  - 权限：公开
  - 返回：String

主要数据结构：LabUserProfileDTO（包含 id、username、realName、gender、identity、academicStatus、phone、email、status、enrollmentYear、graduationYear、photo、resume、homepageUrl、orcid、isActive 等及其描述字段）

### 1.2 实验室用户 CRUD API（LabUserCrudController）
基路径：/lab/users/crud

- POST /
  - 用途：创建用户
  - 权限：lab:user:add 或 管理员
  - 请求体：CreateLabUserCommand
  - 返回：Long（新用户ID）

- PUT /{userId}
  - 用途：管理员更新用户信息
  - 权限：lab:user:edit 或 管理员
  - 请求体：UpdateLabUserCommand（后端会以路径 userId 覆盖命令中的 id）
  - 返回：Void

- PUT /profile
  - 用途：用户更新个人信息
  - 权限：已登录
  - 请求体：UpdateProfileCommand
  - 返回：Void

- PUT /password
  - 用途：用户修改自己的密码
  - 权限：已登录
  - 请求体：ChangePasswordCommand（含旧密码、新密码）
  - 返回：Void

- PUT /{userId}/password
  - 用途：管理员重置用户密码
  - 权限：lab:user:edit 或 管理员
  - 请求体：ResetLabUserPasswordCommand
  - 返回：Void

- GET /list
  - 用途：分页查询用户列表，支持多条件
  - 权限：lab:user:list 或 管理员
  - 查询：LabUserQuery + 分页参数
  - 返回：PageDTO<LabUserProfileDTO>

- GET /search?keyword=...
  - 用途：关键词搜索用户
  - 权限：已登录
  - 返回：List<LabUserProfileDTO>

- DELETE /{userId}
  - 用途：管理员删除用户（硬删）
  - 权限：lab:user:remove 或 管理员
  - 返回：Void

- DELETE /batch
  - 用途：管理员批量删除用户（硬删）
  - 权限：lab:user:remove 或 管理员
  - 请求体：List<Long>（用户ID集合）
  - 返回：Void

- PUT /batch/status?isActive=true|false
  - 用途：批量启用/禁用用户
  - 权限：lab:user:edit 或 管理员
  - 请求体：List<Long>（用户ID集合）
  - 返回：Void（当前实现为占位）

- GET /statistics
  - 用途：获取用户统计信息（占位）
  - 权限：lab:user:list 或 管理员
  - 返回：Map（占位）

- GET /export
  - 用途：导出用户列表（占位）
  - 权限：lab:user:export 或 管理员
  - 返回：String（占位）

- POST /import
  - 用途：导入用户（占位）
  - 权限：lab:user:import 或 管理员
  - 返回：String（占位）

## 模块二：成果管理

说明：管理实验室成果（论文/项目）、作者关系、用户侧“我的成果”、以及相关字典。

### 2.1 成果管理（LabAchievementController）
基路径：/lab/achievements

- GET /
  - 用途：分页查询成果
  - 权限：lab:achievement:list
  - 查询：LabAchievementQuery + 分页参数
  - 返回：PageDTO<LabAchievementDTO>

- GET /{id}
  - 用途：成果详情
  - 权限：lab:achievement:query
  - 返回：LabAchievementDTO

- POST /
  - 用途：创建成果（可选批量附带作者）
  - 权限：lab:achievement:add
  - 请求体：CreateLabAchievementCommand
  - 返回：Long（成果ID）

- PUT /{id}
  - 用途：更新成果
  - 权限：lab:achievement:edit
  - 请求体：CreateLabAchievementCommand（作为更新载体）
  - 返回：Void

- DELETE /{id}
  - 用途：删除成果（应用服务内含权限校验/软删）
  - 权限：lab:achievement:remove；允许角色：管理员 或 拥有者 或 第一作者(内部且author_order=1)
  - 返回：Void

- PUT /{id}/publish?published=true|false
  - 用途：发布/取消发布（隐藏）
  - 权限：
    - 发布（published=true）：仅管理员
    - 隐藏（published=false）：管理员 或 教师且为拥有者/参与者
  - 返回：Void

- PUT /{id}/verify?verified=true|false
  - 用途：审核通过/取消审核
  - 权限：lab:achievement:verify（通常仅管理员）
  - 返回：Void

主要请求体：CreateLabAchievementCommand（关键字段）
- title(必填,<=500)、titleEn、description、keywords(<=1000)
- type(必填：1论文/2项目)
- paperType(1-7，仅type=1生效)、projectType(1-8，仅type=2生效)
- venue(<=300)、publishDate(yyyy)
- projectStartDate(yyyy-MM)、projectEndDate(yyyy-MM)
- coverUrl/linkUrl/gitUrl/homepageUrl/pdfUrl(<=500)
- doi(<=128)、fundingAmount(>=0)、published(Boolean)、extra
- authors: List<CreateAuthorCommand>（可选）

主要查询：LabAchievementQuery
- keyword、type、paperType、projectType、published、isVerified、ownerUserId
- dateStart、dateEnd（按 type 自动匹配 publish_date 或 project_start_date）
- 分页参数（继承 AbstractPageQuery）

### 2.2 成果作者管理（LabAchievementAuthorController）
基路径：/lab/achievements/{achievementId}/authors

- GET /
  - 用途：获取作者列表（按作者顺序）
  - 权限：lab:achievement:query
  - 返回：List<LabAchievementAuthorEntity>

- POST /
  - 用途：新增作者（内部/外部作者均支持）
  - 权限：lab:achievement:edit（且需成果拥有者或管理员）
  - 请求体：AddAuthorCommand（userId 或 外部作者姓名 name 必填其一；authorOrder 唯一）
  - 返回：Long（作者ID）

  - 入参优先级与自动绑定策略：
    - 第一优先：userId（若传且存在对应 lab_user，即绑定为内部作者；同一成果下同一 userId 只允许一次）
    - 第二优先：username/email/phone/studentNumber（按此优先级尝试唯一解析为 lab_user 并绑定）
    - 第三优先：name/nameEn（仅当唯一姓名能命中时才自动绑定；否则作为外部作者 userId 为空）
    - 去重保护：若同一成果下已存在该内部作者，将不会重复绑定
  - 示例：
    - 内部作者（最稳妥，推荐明确传 userId）：
      {"userId":25, "name":"李金峰", "authorOrder":1, "visible":true}
    - 通过 username 自动绑定（未传 userId 时）：
      {"username":"lijinfeng", "name":"李金峰", "authorOrder":1}
    - 外部作者：
      {"name":"合作方A", "authorOrder":2}


- PUT /{authorId}
  - 用途：更新作者
  - 权限：lab:achievement:edit（且需成果拥有者或管理员）
  - 请求体：UpdateAuthorCommand（支持更新 userId/name/nameEn/affiliation/authorOrder/isCorresponding/role/visible）
  - 返回：Void

- DELETE /{authorId}
  - 用途：删除作者（软删）
  - 权限：lab:achievement:edit（且需成果拥有者或管理员）
  - 返回：Void

- PUT /{authorId}/reorder?newOrder=1..
  - 用途：调整作者顺序（需唯一）
  - 权限：lab:achievement:edit（且需成果拥有者或管理员）
  - 返回：Void

- PUT /{authorId}/visibility?visible=true|false
  - 用途：切换个人页可见性（仅内部作者生效）
  - 权限：lab:achievement:edit（且需成果拥有者或管理员）
  - 返回：Void


  - 额外查询：
    - ownerName：拥有者姓名（模糊匹配 real_name/english_name；仅在“我的成果”范围内生效）
    - authorName：作者姓名（模糊匹配作者表 name/name_en 以及内部作者姓名；仅在“我的成果”范围内生效）

校验与权限：
- 编辑权限：管理员 或 成果拥有者（LabAchievementService.isOwner）
- 顺序唯一：authorService.isAuthorOrderExists(achievementId, order, excludeAuthorId)

### 2.3 我的成果（LabMyAchievementController）
基路径：/lab/my-achievements

- GET /
  - 用途：我的成果列表（我拥有的 + 我参与的）
  - 权限：已登录
  - 查询：MyAchievementQuery + 分页参数
  - 返回：PageDTO<LabAchievementDTO>

- PUT /{achievementId}/visibility?visible=true|false
  - 用途：切换“我”在该成果中的个人页可见性（作者本人可操作）
  - 权限：已登录
  - 返回：Void

### 2.4 字典（LabDictController）
基路径：/lab/dicts（公开）

- GET /academic-status
  - 学术身份：0负责人,1教授,2副教授,3讲师,4博士,5硕士,6本科

- GET /achievement-types
  - 成果类型：1论文,2项目

- GET /paper-types
  - 论文类型：1期刊,2会议,3书籍章节,4专利,5标准,6研究报告,7其他论文

- GET /project-types
  - 项目类型：1国家重点,2国家一般,3省部级,4企业合作,5国际合作,6青年基金,7博士后基金,8其他

## 鉴权与角色
- 统一登录入口：POST /login（与原系统一致），JWT 下发后访问上述接口
- lab_user 身份到权限映射（示例）：
  - 1 → lab:admin（默认全权限）
  - 2 → lab:teacher（教学/成果主要权限集）
  - 3 → lab:student（基础权限）
- 关键权限标识：
  - 用户：lab:user:list/query/add/edit/remove/export/import
  - 成果：lab:achievement:list/query/add/edit/remove/publish/verify

## 开发与联调指引
- Swagger：/swagger-ui.html → 选择“实验室管理API”
- 常见错误码：
  - 0 成功；401 未授权；403 权限不足；404 对象不存在；其余见返回 msg
- 建议：前端表单字段与 DTO 对齐；过滤项与 LabAchievementQuery 对齐

## 数据模型（web 库）

下述为与接口直接相关的三张核心表结构要点与约束，便于前后端校验与排错。

### lab_user（实验室用户表）
- 主键/唯一
  - id BIGINT PK
  - 唯一：username, student_number, phone, email
- 关键字段
  - username VARCHAR(50) NOT NULL；password VARCHAR(255) NOT NULL（BCrypt）
  - real_name VARCHAR(50) NOT NULL
  - identity TINYINT NOT NULL（1管理员/2教师/3学生）
  - gender TINYINT 默认 0（0未知/1男/2女）
  - academic_status TINYINT 可空（字典提供枚举）
  - email/phone UNIQUE 可空；student_number UNIQUE 可空
  - status TINYINT 默认 1（1在读/在职，2毕业/离职）
  - enrollment_year/graduation_year YEAR，可空
  - resume/homepage_url/orcid 等个人信息
  - is_active TINYINT(1) 默认 1；deleted TINYINT(1) 默认 0
- 索引
  - (identity, academic_status)、enrollment_year 等

接口注意：创建/更新需处理 username/email/phone/student_number 唯一冲突；is_active 控制登录可用性。

### lab_achievement（实验室成果表）
- 主键/外键/检查
  - id BIGINT PK；owner_user_id BIGINT NOT NULL → lab_user(id)
  - 检查：funding_amount >= 0；按 type 校验字段组合
    - type=1 论文：paper_type(1-7)、publish_date NOT NULL；项目日期字段需 NULL
    - type=2 项目：project_type(1-8)、project_start_date NOT NULL；project_end_date 为 NULL 或 >= start
- 关键字段
  - title VARCHAR(500) NOT NULL；title_en/description/keywords
  - type TINYINT NOT NULL（1论文/2项目）
  - paper_type / project_type
  - venue VARCHAR(300)
  - publish_date（论文）、project_start_date/project_end_date（项目）
  - cover_url/link_url/git_url/homepage_url/pdf_url（≤500）、doi（≤128）
  - funding_amount DECIMAL(12,2)
  - published TINYINT(1) 默认 0；is_verified TINYINT(1) 默认 0
  - extra JSON；deleted TINYINT(1) 默认 0
- 索引
  - (type, publish_date)、(type, project_start_date, project_end_date)
  - (published, publish_date)、owner_user_id、doi、deleted

接口注意：创建/更新须满足组合校验；发布/审核通常仅管理员；公开列表常用 published=1。

### lab_achievement_author（成果作者表）
- 主键/外键/唯一/检查
  - id BIGINT PK
  - achievement_id BIGINT NOT NULL → lab_achievement(id) ON DELETE CASCADE
  - user_id BIGINT 可空 → lab_user(id)（内部作者）
  - UNIQUE (achievement_id, author_order, deleted)（未删条件下顺序唯一）
  - UNIQUE (achievement_id, user_id)（内部作者在该成果仅出现一次）
  - 检查：author_order > 0；(user_id 非空) 或 (name 非空)
- 关键字段
  - name VARCHAR(100) NOT NULL（外部作者必填；内部作者可冗余）
  - name_en, affiliation；author_order INT NOT NULL
  - is_corresponding TINYINT(1) 默认 0；role VARCHAR(100)
  - visible TINYINT(1) 默认 1（仅内部作者生效：个人页是否展示）
  - deleted TINYINT(1) 默认 0
- 索引
  - achievement_id、user_id、(user_id, visible, author_order)、(achievement_id, deleted)

接口注意：新增/更新需保证 author_order 唯一；内部作者不可重复；删除为逻辑删除且唯一约束含 deleted 避免冲突。
