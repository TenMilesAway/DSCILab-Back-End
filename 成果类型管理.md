# 成果类型管理模块设计方案

## 一、需求分析

### 1.1 现状分析
- 当前成果类型：固定的论文(1)和项目(2)
- 论文子类型：1-7种固定类型
- 项目子类型：1-8种固定类型
- 类型信息硬编码在枚举中

### 1.2 目标需求
- 支持动态的一级类型管理（论文、项目 + 新增类型）
- 支持每个一级类型下的二级类型管理
- 管理员可以增删改查类型
- 保持向下兼容，不影响现有成果数据

## 二、数据库设计

### 2.1 新增表：lab_achievement_category（成果类型表）

```sql
CREATE TABLE `lab_achievement_category` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '类型ID',
  `parent_id` BIGINT NULL COMMENT '父类型ID，NULL表示一级类型',
  `category_code` VARCHAR(50) NOT NULL COMMENT '类型编码（用于兼容现有逻辑）',
  `category_name` VARCHAR(100) NOT NULL COMMENT '类型名称',
  `category_name_en` VARCHAR(100) NULL COMMENT '英文名称',
  `description` TEXT NULL COMMENT '类型描述',
  `sort_order` INT DEFAULT 0 COMMENT '排序顺序',
  `is_system` BOOLEAN DEFAULT FALSE COMMENT '是否系统内置类型（不可删除）',
  `is_active` BOOLEAN DEFAULT TRUE COMMENT '是否启用',
  `icon` VARCHAR(100) NULL COMMENT '图标',
  `color` VARCHAR(20) NULL COMMENT '颜色标识',
  
  -- 审计字段
  `creator_id` BIGINT NULL COMMENT '创建者ID',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updater_id` BIGINT NULL COMMENT '更新者ID', 
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` BOOLEAN DEFAULT FALSE COMMENT '逻辑删除标记',
  
  -- 索引
  KEY `idx_parent_id` (`parent_id`),
  KEY `idx_category_code` (`category_code`),
  KEY `idx_sort_order` (`sort_order`),
  KEY `idx_is_active` (`is_active`),
  KEY `idx_deleted` (`deleted`),
  
  -- 唯一约束
  UNIQUE KEY `uk_code_deleted` (`category_code`, `deleted`),
  UNIQUE KEY `uk_parent_name_deleted` (`parent_id`, `category_name`, `deleted`),
  
  -- 外键约束
  CONSTRAINT `fk_category_parent` FOREIGN KEY (`parent_id`) REFERENCES `lab_achievement_category` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='成果类型表';
```

## 五、兼容性设计

### 5.1 数据兼容策略

#### 5.1.1 现有成果数据兼容
- 保持 `lab_achievement` 表的 `type`、`paper_type`、`project_type` 字段不变
- 新增字段 `category_id`（可为空，用于关联新的类型系统）
- 提供数据迁移工具，将现有数据映射到新的类型系统

#### 5.1.2 映射关系
```
现有type=1, paper_type=1 → category_id=3 (期刊论文)
现有type=1, paper_type=2 → category_id=4 (会议论文)
现有type=2, project_type=1 → category_id=10 (国家重点项目)
...
```

### 5.2 代码兼容策略

#### 5.2.1 兼容性服务类
```java
@Service
public class CategoryCompatibilityService {
    
    /**
     * 根据旧的type和subType获取新的categoryId
     */
    public Long getCategoryIdByLegacyType(Integer type, Integer subType);
    
    /**
     * 根据categoryId获取旧的type和subType
     */
    public LegacyTypeMapping getLegacyTypeByCategory(Long categoryId);
    
    /**
     * 迁移现有成果数据到新类型系统
     */
    public void migrateExistingAchievements();
}
```

## 六、业务逻辑设计

### 6.1 类型管理业务规则

#### 6.1.1 创建规则
- 类型编码全局唯一
- 同一父类型下的类型名称唯一
- 最多支持2级分类（一级类型 + 二级类型）
- 排序号在同级类型中唯一

#### 6.1.2 删除规则
- 系统内置类型不可删除
- 有子类型的类型不可删除
- 被成果引用的类型不可删除
- 删除时提供数据迁移建议

#### 6.1.3 排序规则
- 支持拖拽排序
- 同级类型间排序
- 排序号自动重新计算

### 6.2 权限控制

#### 6.2.1 权限定义
```
lab:category:list    # 查看类型列表
lab:category:query   # 查看类型详情  
lab:category:add     # 创建类型
lab:category:edit    # 编辑类型
lab:category:remove  # 删除类型
lab:category:sort    # 排序类型
```

#### 6.2.2 角色权限映射
- 管理员：所有权限
- 教师：查看权限
- 学生：查看权限

### 8.3 更新类型
```
PUT /lab/achievement-categories/{id}
权限：lab:category:edit 或 管理员

请求体：同创建接口，所有字段可选

业务规则：
- 系统内置类型（is_system=true）不允许删除，但可以修改名称和描述
- 不能将类型移动到自己的子类型下（避免循环引用）
- 修改父类型时需要检查是否会造成循环引用
```

### 8.4 删除类型
```
DELETE /lab/achievement-categories/{id}
权限：lab:category:remove 或 管理员

业务规则：
- 系统内置类型不允许删除
- 有子类型的类型不允许删除
- 已被成果使用的类型不允许删除（需要先迁移成果）
- 执行软删除
```

## 九、数据迁移方案

### 9.1 迁移步骤
1. 创建新表并插入初始数据
2. 为 `lab_achievement` 表添加 `category_id` 字段
3. 执行数据迁移脚本
4. 验证迁移结果
5. 更新应用代码

### 9.2 迁移脚本示例
```sql
-- 添加新字段
ALTER TABLE `lab_achievement` ADD COLUMN `category_id` BIGINT NULL COMMENT '成果类型ID' AFTER `project_type`;
ALTER TABLE `lab_achievement` ADD KEY `idx_category_id` (`category_id`);

-- 数据迁移
UPDATE `lab_achievement` SET `category_id` = 3 WHERE `type` = 1 AND `paper_type` = 1;
UPDATE `lab_achievement` SET `category_id` = 4 WHERE `type` = 1 AND `paper_type` = 2;
-- ... 其他映射关系

-- 添加外键约束（可选）
ALTER TABLE `lab_achievement` ADD CONSTRAINT `fk_achievement_category`
FOREIGN KEY (`category_id`) REFERENCES `lab_achievement_category` (`id`);
```
