# 实验室管理系统整体框架与方法总结

## 一、系统架构概览

### 1.1 整体架构
基于 Spring Boot + MyBatis-Plus 的分层架构，采用 DDD（领域驱动设计）思想：

```
agileboot-admin/          # 控制器层（Web接口）
├── controller/
│   ├── common/          # 通用控制器（登录、文件上传等）
│   ├── lab/             # 实验室模块控制器
│   ├── open/            # 公开接口控制器
│   └── system/          # 系统管理控制器

agileboot-domain/         # 领域层（核心业务逻辑）
├── lab/                 # 实验室领域
│   ├── user/           # 用户管理子域
│   └── achievement/    # 成果管理子域
└── system/             # 系统管理领域

agileboot-infrastructure/ # 基础设施层
agileboot-common/         # 通用工具层
```

### 1.2 技术栈
- **框架**: Spring Boot 2.7+, Spring Security, MyBatis-Plus
- **数据库**: MySQL 8.0+
- **认证**: JWT + Spring Security
- **文档**: Swagger/OpenAPI 3.0
- **测试**: JUnit 5 + Mockito + MockMvc
- **构建**: Maven

## 二、实验室用户管理模块

### 2.1 核心实体
**LabUserEntity** - 实验室用户实体
- 基础信息：学号/工号、用户名、真实姓名、英文名
- 身份信息：性别、身份（管理员/教师/学生）、学术身份
- 联系信息：手机、邮箱、地址
- 学术信息：研究方向、个人简介、主页、ORCID
- 系统信息：账号状态、创建/更新时间

### 2.2 身份与权限体系
**身份枚举**:
- ADMIN(1, "管理员")
- TEACHER(2, "教师") 
- STUDENT(3, "学生")

**学术身份枚举**:
- LAB_DIRECTOR(0, "实验室负责人")
- PROFESSOR(1, "教授")
- ASSOCIATE_PROFESSOR(2, "副教授")
- LECTURER(3, "讲师")
- PHD(4, "博士")
- MASTER(5, "硕士")
- UNDERGRADUATE(6, "本科")

### 2.3 服务层架构
**LabUserService** - 基础服务层
- getByUsername(String username)
- getByStudentNumber(String studentNumber)
- getByEmail(String email)
- getByPhone(String phone)
- isUsernameDuplicated(String username, Long excludeId)
- isStudentNumberDuplicated(String studentNumber, Long excludeId)
- isEmailDuplicated(String email, Long excludeId)
- isPhoneDuplicated(String phone, Long excludeId)

**LabUserApplicationService** - 应用服务层
- getCurrentUserProfile()
- getUserProfile(Long userId)
- getUserProfileByUsername(String username)
- userExists(Long userId)

**LabUserCrudApplicationService** - CRUD应用服务
- getLabUserList(LabUserQuery query)
- getLabUserDetail(Long id)
- createLabUser(CreateLabUserCommand command)
- updateLabUser(Long id, UpdateLabUserCommand command)
- deleteLabUser(Long id)
- changePassword(Long id, ChangePasswordCommand command)

### 2.4 控制器层
**LabUserController** - 用户信息查询
- GET /lab/users/profile - 获取当前用户信息
- GET /lab/users/{userId} - 根据ID获取用户信息
- GET /lab/users/username/{username} - 根据用户名获取信息

**LabUserCrudController** - 用户CRUD管理
- GET /lab/users - 用户列表（分页、筛选）
- GET /lab/users/{id} - 用户详情
- POST /lab/users - 创建用户
- PUT /lab/users/{id} - 更新用户
- DELETE /lab/users/{id} - 删除用户
- PUT /lab/users/{id}/password - 修改密码

### 2.5 权限控制
**LabUserPermissionChecker** - 权限检查器
- getCurrentLabUser() - 获取当前实验室用户
- isAdmin() - 是否管理员
- canViewUser(Long userId) - 是否可查看用户
- canViewUserByUsername(String username) - 是否可查看用户

**权限注解**:
- @PreAuthorize("@permission.has('lab:user:list')")
- @PreAuthorize("@permission.has('lab:user:query')")
- @PreAuthorize("@permission.has('lab:user:add')")
- @PreAuthorize("@permission.has('lab:user:edit')")
- @PreAuthorize("@permission.has('lab:user:remove')")

## 三、成果管理模块

### 3.1 核心实体
**LabAchievementEntity** - 成果实体
- 基础信息：标题、英文标题、描述、关键词
- 类型信息：成果类型（论文/项目）、论文类型、项目类型
- 发表信息：发表场所、发表日期、项目时间、经费
- 链接信息：封面、链接、Git、主页、PDF、DOI
- 状态信息：拥有者、发布状态、审核状态、删除状态

**LabAchievementAuthorEntity** - 作者实体
- 关联信息：成果ID、用户ID（内部作者）
- 作者信息：姓名、英文名、单位
- 排序信息：作者顺序、是否通讯作者、角色
- 可见性：是否在个人页显示（仅内部作者）

### 3.2 服务层架构
**LabAchievementService** - 成果基础服务
- getByIdNotDeleted(Long id)
- canEdit(Long achievementId, Long userId, boolean isAdmin)
- isOwner(Long achievementId, Long userId)

**LabAchievementAuthorService** - 作者基础服务
- getAuthorsByAchievementId(Long achievementId)
- isAuthor(Long achievementId, Long userId)
- getAuthorRecord(Long achievementId, Long userId)
- isAuthorOrderExists(Long achievementId, Integer order, Long excludeId)

**LabAchievementApplicationService** - 成果应用服务
- getAchievementList(LabAchievementQuery query)
- getAchievementDetail(Long id)
- createAchievement(CreateLabAchievementCommand command, Long currentUserId)
- updateAchievement(Long id, CreateLabAchievementCommand command, Long currentUserId, boolean isAdmin)
- deleteAchievement(Long id, Long currentUserId, boolean isAdmin)
- publishAchievement(Long id, Boolean published, Long currentUserId, boolean isAdmin)
- verifyAchievement(Long id, Boolean verified, Long currentUserId, boolean isAdmin)
- getMyAchievementList(MyAchievementQuery query, Long currentUserId)
- getPublicAchievementList(PublicAchievementQuery query)
- getPublicAchievementDetail(Long id)
- toggleMyVisibilityInAchievement(Long achievementId, Boolean visible, Long currentUserId)

### 3.3 控制器层
**LabAchievementController** - 成果管理（管理员/拥有者）
- GET /lab/achievements - 成果列表
- GET /lab/achievements/{id} - 成果详情
- POST /lab/achievements - 创建成果（支持批量作者）
- PUT /lab/achievements/{id} - 更新成果
- DELETE /lab/achievements/{id} - 删除成果
- PUT /lab/achievements/{id}/publish - 发布/取消发布
- PUT /lab/achievements/{id}/verify - 审核/取消审核

**LabAchievementAuthorController** - 作者管理
- GET /lab/achievements/{achievementId}/authors - 作者列表
- POST /lab/achievements/{achievementId}/authors - 新增作者
- PUT /lab/achievements/{achievementId}/authors/{authorId} - 更新作者
- DELETE /lab/achievements/{achievementId}/authors/{authorId} - 删除作者
- PUT /lab/achievements/{achievementId}/authors/{authorId}/reorder - 调整顺序
- PUT /lab/achievements/{achievementId}/authors/{authorId}/visibility - 切换可见性

**LabMyAchievementController** - 用户自助端
- GET /lab/my-achievements - 我的成果列表
- PUT /lab/my-achievements/{achievementId}/visibility - 切换个人可见性

**OpenAchievementController** - 公开端（无需登录）
- GET /open/achievements - 公开成果列表
- GET /open/achievements/{id} - 公开成果详情

### 3.4 权限控制
**成果权限**:
- lab:achievement:list - 成果列表查询
- lab:achievement:query - 成果详情查询
- lab:achievement:add - 创建成果
- lab:achievement:edit - 编辑成果
- lab:achievement:remove - 删除成果
- lab:achievement:publish - 发布成果
- lab:achievement:verify - 审核成果

**业务权限规则**:
- 管理员：所有操作权限
- 拥有者：可编辑/删除自己的成果
- 作者：可编辑参与的成果（部分字段）
- 发布/审核：仅管理员

## 九、核心设计模式

### 9.1 分层架构
- **Controller**: 接口层，处理HTTP请求
- **ApplicationService**: 应用服务层，编排业务流程
- **Service**: 领域服务层，核心业务逻辑
- **Entity**: 实体层，数据模型和业务规则
- **Mapper**: 数据访问层，数据库操作

### 9.2 CQRS模式
- **Command**: 写操作命令对象
- **Query**: 读操作查询对象
- **DTO**: 数据传输对象

### 9.3 权限控制模式
- **基于注解的权限控制**: @PreAuthorize
- **业务权限检查**: PermissionChecker
- **多级权限验证**: 系统权限 + 业务权限

### 9.4 软删除模式
- **@TableLogic**: MyBatis-Plus 逻辑删除
- **联动软删**: 删除成果时联动软删作者
- **查询过滤**: 自动过滤已删除记录
