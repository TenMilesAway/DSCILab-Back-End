# 对外展示接口汇总

本文汇总“成员管理 / 论文管理 / 项目管理”在门户前端使用的公开接口，方便修改前端展示页时统一参考。所有接口均无需登录认证，默认只返回已发布或启用的数据。

---

## 1. 成员展示（OpenLabController）

### 1.1 公开成员列表
- **方法 & 地址**：`GET /open/users`
- **参数**：
  | 参数 | 说明 |
  | --- | --- |
  | `pageNum` / `pageSize` | 默认 1 / 1000，可自行分页 |
  | `identity` | 1=管理员、2=教师、3=学生 |
  | `academicStatus` | 1=教授、2=副教授、3=讲师、4=博士、5=硕士等 |
  | `status` | 1=在读/在职，2=离职/毕业 |
  | `keyword` | 按 `realName/englishName/researchArea` 模糊搜索 |
- **响应**：`ResponseDTO<PageDTO<PublicLabUserDTO>>`
  - PublicLabUserDTO 字段：`id`、`studentNumber`、`realName/englishName`、`gender`、`identity`、`academicStatus`、`researchArea`、`phone`、`status`、`enrollmentYear`、`graduationYear`、`graduationDest`、`photo`、`resume`、`homepageUrl`、`email`、`orcid`。
- **缓存**：服务器返回 `Cache-Control: public, max-age=60`。

### 1.2 公开成员详情
- **方法 & 地址**：`GET /open/users/{id}`
- **说明**：当用户不存在、被删除或 `isActive=false` 时返回 `data=null`，其余字段与列表一致。
- **缓存**：同样 60 秒公共缓存。

## 2. 论文展示（OpenPaperController）

### 2.1 公开论文列表
- **方法 & 地址**：`GET /open/papers`
- **参数（PublicPaperQuery）**：
  | 参数 | 说明 |
  | --- | --- |
  | `pageNum` / `pageSize` | 分页 |
  | `keyword` | 标题 / 作者文本 / 关键词 模糊匹配 |
  | `paperTypeId` | 1~7（期刊/会议等） |
  | `categoryId` / `parentCategoryId` | 成果分类筛选 |
  | `dateStart` / `dateEnd` | 发表日期范围（YYYY-MM-DD） |
  | `ownerUserId` |（可选）按成果所有者过滤 |
- **响应**：`PageDTO<LabPaperDTO>`
  - 核心字段：`title/titleEn/description/keywords/publication/conferenceName/issueNumber/publisher/publishDate`。
  - 链接：`linkUrl/extraUrl/gitUrl/homepageUrl/pdfUrl`。
  - 关联信息：
    - `authors`（按 `authorOrder` 排序，含 `isCorresponding`、`affiliation`、`role`、`visible`）。
    - `relatedProjects`（项目 ID/标题/分类/时间范围）。
    - `fundAssociations`（基金 ID、金额）。
  - 分类：`categoryId/categoryName/categoryFullPath`、`typeInfo`（兼容旧枚举）。

### 2.2 公开论文详情
- **方法 & 地址**：`GET /open/papers/{id}`
- **说明**：未发布的论文直接返回 404；已发布的详情与列表字段一致，适用于详情页展示。

## 3. 项目展示（OpenProjectController）

### 3.1 公开项目列表
- **方法 & 地址**：`GET /open/projects`
- **参数（LabProjectQuery）**：
  | 参数 | 说明 |
  | --- | --- |
  | `pageNum` / `pageSize` | 分页 |
  | `keyword` | 标题 / 关键词 模糊匹配 |
  | `projectTypeId` | 1~8（校级/部级等） |
  | `supporter` | 资助单位模糊 |
  | `categoryId` / `parentCategoryId` | 成果分类筛选 |
  | `startDateBegin` / `startDateEnd` | 项目开始日期范围 |
- **响应**：`PageDTO<LabProjectDTO>`
  - 基础字段：`title/titleEn/description/keywords/projectNumber/supporter/projectStartDate/projectEndDate/fundingAmount/amountDisplay/supportCn/supportEn/requirement/descText/reference`。
  - 链接：`linkUrl/gitUrl/homepageUrl/pdfUrl`。
  - 作者：`authors`（沿用 `LabPaperAuthorDTO`，含姓名/机构/通讯作者标记）。
  - 关联论文：`relatedPapers`（论文 ID/标题/类型/分类/发表日期）。
  - 分类：`categoryId/categoryName/categoryFullPath`、`projectTypeId` 对应描述。

### 3.2 公开项目详情
- **方法 & 地址**：`GET /open/projects/{id}`
- **说明**：仅 `published=true` 的项目返回，字段与列表一致，适合详情页展示。

---

## 4. 使用建议
1. **成员页**：调用 `/open/users` 做卡片式展示，详情使用 `/open/users/{id}`；头像、研究方向、主页等字段可直接渲染。
2. **论文页**：列表页结合 `keyword`、`categoryId`、`paperTypeId` 等参数筛选；详情页可展示作者信息、基金、关联项目，支持在 `relatedProjects` 上跳转。
3. **项目页**：列表页按 `projectTypeId`、`supporter` 或时间范围过滤；详情页可展示关联论文与作者，若需要突出分类，可以使用 `categoryFullPath`。

如需扩展展示字段，可在对应 DTO（`PublicLabUserDTO`、`LabPaperDTO`、`LabProjectDTO`）中查阅现有可用属性，或在领域层补充新的投影后再暴露给前端。
