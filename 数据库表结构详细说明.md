## 1. lab_user - 实验室用户表

### 表说明
存储实验室成员的基本信息，包括教师、学生、管理员等。

### 字段详情

| 字段名 | 类型 | 长度 | 是否必填 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | BIGINT | - | 是 | AUTO_INCREMENT | 主键，用户ID |
| student_number | VARCHAR | 20 | 否 | NULL | 学号/工号，唯一标识 |
| username | VARCHAR | 50 | 是 | - | 登录用户名，唯一 |
| real_name | VARCHAR | 100 | 是 | - | 真实姓名 |
| english_name | VARCHAR | 100 | 否 | NULL | 英文姓名 |
| password | VARCHAR | 255 | 是 | - | 密码（BCrypt加密） |
| gender | TINYINT | - | 否 | 0 | 性别：0=未知,1=男,2=女 |
| identity | TINYINT | - | 是 | 3 | 身份：1=管理员,2=教师,3=学生 |
| academic_status | TINYINT | - | 否 | NULL | 学术身份：0=实验室负责人,1=教授,2=副教授,3=讲师,4=博士,5=硕士,6=本科 |
| research_area | TEXT | - | 否 | NULL | 研究方向 |
| phone | VARCHAR | 20 | 否 | NULL | 手机号 |
| email | VARCHAR | 100 | 否 | NULL | 邮箱地址 |
| status | TINYINT | - | 是 | 1 | 状态：1=在读/在职,2=毕业/离职 |
| enrollment_year | YEAR | - | 否 | NULL | 入学/入职年份 |
| graduation_year | YEAR | - | 否 | NULL | 毕业/离职年份 |
| graduation_dest | VARCHAR | 500 | 否 | NULL | 毕业去向 |
| photo | VARCHAR | 500 | 否 | NULL | 照片路径 |
| resume | TEXT | - | 否 | NULL | 个人简历 |
| homepage_url | VARCHAR | 500 | 否 | NULL | 个人主页URL |
| orcid | VARCHAR | 50 | 否 | NULL | ORCID ID |
| is_active | BOOLEAN | - | 是 | TRUE | 账号是否启用 |
| deleted | BOOLEAN | - | 是 | FALSE | 逻辑删除标记 |
| creator_id | BIGINT | - | 否 | NULL | 创建者ID |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updater_id | BIGINT | - | 否 | NULL | 更新者ID |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 更新时间 |

### 枚举值说明

#### 身份类型 (identity)
- 1: 管理员 - 拥有系统最高权限
- 2: 教师 - 可以管理自己的成果和学生
- 3: 学生 - 基本用户权限

#### 学术身份 (academic_status)
- 0: 实验室负责人
- 1: 教授
- 2: 副教授
- 3: 讲师
- 4: 博士研究生
- 5: 硕士研究生
- 6: 本科生

#### 状态 (status)
- 1: 在读/在职 - 当前活跃成员
- 2: 毕业/离职 - 已离开实验室

### 唯一约束
- username: 登录用户名唯一
- student_number: 学号/工号唯一（非空时）
- phone: 手机号唯一（非空时）
- email: 邮箱唯一（非空时）

## 2. lab_achievement - 实验室成果表

### 表说明
存储实验室的研究成果信息，包括论文、项目等。

### 字段详情

| 字段名 | 类型 | 长度 | 是否必填 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | BIGINT | - | 是 | AUTO_INCREMENT | 主键，成果ID |
| title | VARCHAR | 500 | 是 | - | 成果标题 |
| title_en | VARCHAR | 500 | 否 | NULL | 英文标题 |
| description | TEXT | - | 否 | NULL | 摘要/描述 |
| keywords | VARCHAR | 1000 | 否 | NULL | 关键词 |
| type | TINYINT | - | 是 | - | 成果类型：1=论文,2=项目 |
| paper_type | TINYINT | - | 否 | NULL | 论文类型（1-7） |
| project_type | TINYINT | - | 否 | NULL | 项目类型（1-8） |
| venue | VARCHAR | 300 | 否 | NULL | 期刊/会议/发布渠道 |
| publish_date | DATE | - | 否 | NULL | 发表/公开日期（论文用） |
| project_start_date | DATE | - | 否 | NULL | 项目开始日期 |
| project_end_date | DATE | - | 否 | NULL | 项目结束日期 |
| reference | TEXT | - | 否 | NULL | 参考文献/引用信息 |
| link_url | VARCHAR | 500 | 否 | NULL | 外部链接URL |
| git_url | VARCHAR | 500 | 否 | NULL | Git仓库地址 |
| homepage_url | VARCHAR | 500 | 否 | NULL | 主页/展示页地址 |
| pdf_url | VARCHAR | 500 | 否 | NULL | PDF下载地址 |
| doi | VARCHAR | 128 | 否 | NULL | 论文DOI |
| funding_amount | DECIMAL | 10,2 | 否 | NULL | 项目经费（万元） |
| owner_user_id | BIGINT | - | 是 | - | 成果所有者（创建者）ID |
| published | BOOLEAN | - | 是 | FALSE | 是否对外发布 |
| is_verified | BOOLEAN | - | 是 | FALSE | 是否已审核 |
| extra | TEXT | - | 否 | NULL | 扩展信息（JSON格式） |
| deleted | BOOLEAN | - | 是 | FALSE | 逻辑删除标记 |
| creator_id | BIGINT | - | 否 | NULL | 创建者ID |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updater_id | BIGINT | - | 否 | NULL | 更新者ID |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 更新时间 |

### 枚举值说明

#### 成果类型 (type)
- 1: 论文 - 学术论文、期刊文章等
- 2: 项目 - 科研项目、工程项目等

#### 论文类型 (paper_type) - 仅当type=1时有效
- 1: 期刊论文
- 2: 会议论文
- 3: 书籍章节
- 4: 专利
- 5: 标准
- 6: 研究报告
- 7: 其他论文

#### 项目类型 (project_type) - 仅当type=2时有效
- 1: 国家重点项目
- 2: 国家一般项目
- 3: 省部级项目
- 4: 企业合作项目
- 5: 国际合作项目
- 6: 青年基金
- 7: 博士后基金
- 8: 其他项目

### 外键关系
- owner_user_id → lab_user.id (成果所有者)

## 3. lab_achievement_author - 成果作者关联表

### 表说明
存储成果与作者的关联关系，支持内部作者（实验室成员）和外部作者。

### 字段详情

| 字段名 | 类型 | 长度 | 是否必填 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | BIGINT | - | 是 | AUTO_INCREMENT | 主键，关联ID |
| achievement_id | BIGINT | - | 是 | - | 成果ID |
| user_id | BIGINT | - | 否 | NULL | 内部作者用户ID，NULL表示外部作者 |
| name | VARCHAR | 100 | 否 | NULL | 作者姓名（外部作者必填，内部作者可冗余） |
| name_en | VARCHAR | 100 | 否 | NULL | 英文姓名 |
| affiliation | VARCHAR | 300 | 否 | NULL | 单位/机构 |
| author_order | INT | - | 是 | - | 作者顺序（从1开始） |
| is_corresponding | BOOLEAN | - | 是 | FALSE | 是否通讯作者 |
| role | VARCHAR | 100 | 否 | NULL | 作者角色/贡献 |
| visible | BOOLEAN | - | 是 | TRUE | 是否在该用户个人页可见（仅内部作者有效） |
| deleted | BOOLEAN | - | 是 | FALSE | 逻辑删除标记 |
| creator_id | BIGINT | - | 否 | NULL | 创建者ID |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updater_id | BIGINT | - | 否 | NULL | 更新者ID |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 更新时间 |

### 作者类型说明

#### 内部作者 (user_id 不为空)
- 关联到 lab_user 表的实验室成员
- name 字段可以冗余存储，便于查询和显示
- visible 字段控制是否在个人主页显示该成果
- 支持自动绑定：通过姓名、邮箱、手机号等自动识别内部作者

#### 外部作者 (user_id 为空)
- 不属于实验室的合作者
- name 字段必填
- visible 字段无效
- 完全依赖手动录入信息

### 外键关系
- achievement_id → lab_achievement.id (成果)
- user_id → lab_user.id (内部作者，可为空)

### 唯一约束
- (achievement_id, user_id): 同一成果下同一内部作者只能出现一次
- (achievement_id, author_order): 同一成果下作者顺序唯一

## 4. lab_user_relationship - 师生关系表

### 表说明
存储实验室师生关系信息，用于管理导师与学生之间的指导关系。

### 字段详情

| 字段名 | 类型 | 长度 | 是否必填 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | BIGINT | - | 是 | AUTO_INCREMENT | 主键，关系ID |
| teacher_id | BIGINT | - | 是 | - | 导师ID，外键关联lab_user.id |
| student_id | BIGINT | - | 是 | - | 学生ID，外键关联lab_user.id |
| relationship_type | TINYINT | - | 否 | 1 | 关系类型：1=导师关系 |
| start_date | DATE | - | 否 | NULL | 关系开始时间 |
| end_date | DATE | - | 否 | NULL | 关系结束时间（毕业等） |
| status | TINYINT | - | 是 | 1 | 状态：1=活跃,2=已结束 |
| remark | VARCHAR | 255 | 否 | NULL | 备注信息 |
| creator_id | BIGINT | - | 否 | NULL | 创建者ID |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updater_id | BIGINT | - | 否 | NULL | 更新者ID |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 更新时间 |
| deleted | TINYINT(1) | - | 是 | 0 | 逻辑删除标记 |

### 枚举值说明

#### 关系类型 (relationship_type)
- 1: 导师关系 - 导师指导学生的关系

#### 状态 (status)
- 1: 活跃 - 当前有效的师生关系
- 2: 已结束 - 已结束的师生关系（如学生毕业）

### 外键关系
- teacher_id → lab_user.id (导师)
- student_id → lab_user.id (学生)

### 唯一约束
- (teacher_id, student_id, status): 同一对师生在相同状态下只能有一条记录

### 业务规则
1. **一对多关系**: 一个导师可以指导多个学生
2. **一对一限制**: 一个学生同时只能有一个活跃的导师
3. **身份验证**: teacher_id 对应的用户身份必须是教师(identity=2)或管理员(identity=1)
4. **身份验证**: student_id 对应的用户身份必须是学生(identity=3)
5. **状态管理**: 更换导师时，原关系状态变为"已结束"，新关系状态为"活跃"
6. **软删除**: 支持逻辑删除，保留历史关系记录

## 5. 表关系说明

### 主要关系
```
lab_user (1) ←→ (N) lab_achievement
  ↑                    ↓
  │                    │
  │                    ↓
  └─ (0..N) lab_achievement_author (N) → (1) lab_achievement

lab_user (1) ←→ (N) lab_user_relationship
  ↑                    ↓
  │                    │
  └─ (teacher_id)      └─ (student_id)
```

### 关系详情
1. **用户与成果**: 一个用户可以拥有多个成果（owner_user_id）
2. **成果与作者**: 一个成果可以有多个作者，一个用户可以是多个成果的作者
3. **内部作者**: lab_achievement_author.user_id 关联到 lab_user.id
4. **外部作者**: lab_achievement_author.user_id 为空，仅存储姓名等基本信息
5. **师生关系**: 一个导师可以指导多个学生，一个学生同时只能有一个活跃导师
6. **用户角色**: lab_user_relationship 通过 teacher_id 和 student_id 都关联到 lab_user.id

## 5. 索引设计

### lab_user 表索引
```sql
-- 主键索引
PRIMARY KEY (id)

-- 唯一索引
UNIQUE KEY uk_username (username)
UNIQUE KEY uk_student_number (student_number)
UNIQUE KEY uk_phone (phone)
UNIQUE KEY uk_email (email)

-- 普通索引
KEY idx_identity (identity)
KEY idx_status (status)
KEY idx_deleted (deleted)
```

### lab_achievement 表索引
```sql
-- 主键索引
PRIMARY KEY (id)

-- 外键索引
KEY fk_owner_user (owner_user_id)

-- 业务索引
KEY idx_type (type)
KEY idx_published (published)
KEY idx_verified (is_verified)
KEY idx_deleted (deleted)
KEY idx_create_time (create_time)

-- 复合索引
KEY idx_type_published (type, published)
KEY idx_owner_deleted (owner_user_id, deleted)
```

### lab_achievement_author 表索引
```sql
-- 主键索引
PRIMARY KEY (id)

-- 唯一索引
UNIQUE KEY uniq_achievement_user (achievement_id, user_id)
UNIQUE KEY uniq_achievement_order (achievement_id, author_order)

-- 外键索引
KEY fk_achievement (achievement_id)
KEY fk_user (user_id)

-- 业务索引
KEY idx_deleted (deleted)
KEY idx_visible (visible)

-- 复合索引
KEY idx_achievement_deleted (achievement_id, deleted)
KEY idx_user_visible (user_id, visible, deleted)
```

### lab_user_relationship 表索引
```sql
-- 主键索引
PRIMARY KEY (id)

-- 外键索引
KEY fk_teacher (teacher_id)
KEY fk_student (student_id)

-- 业务索引
KEY idx_status (status)
KEY idx_deleted (deleted)
KEY idx_start_date (start_date)
KEY idx_end_date (end_date)

-- 复合索引
KEY idx_teacher_status (teacher_id, status, deleted)
KEY idx_student_status (student_id, status, deleted)
KEY idx_relationship_type (relationship_type, status)

-- 唯一约束索引
UNIQUE KEY uniq_teacher_student_status (teacher_id, student_id, status, deleted)
```

## 6. 业务规则

### 用户管理规则
1. **唯一性**: username、student_number、phone、email 必须唯一
2. **身份权限**: 管理员 > 教师 > 学生
3. **状态管理**: 毕业/离职用户保留数据但标记状态
4. **软删除**: 删除用户时仅标记 deleted=true，保留历史数据

### 成果管理规则
1. **所有权**: 每个成果有唯一的所有者（owner_user_id）
2. **类型约束**: type=1时使用paper_type，type=2时使用project_type
3. **发布控制**: published 控制是否对外展示
4. **审核流程**: is_verified 控制审核状态
5. **软删除**: 删除成果时级联软删除相关作者记录

### 作者管理规则
1. **内外部区分**: user_id 为空表示外部作者
2. **顺序唯一**: 同一成果下 author_order 必须唯一
3. **可见性控制**: 内部作者可通过 visible 字段控制个人页面显示
4. **自动绑定**: 支持通过多种方式自动识别内部作者
5. **软删除复活**: 删除后重新添加同一内部作者会复活原记录

### 师生关系管理规则
1. **导师资格**: 只有身份为教师(identity=2)或管理员(identity=1)的用户才能作为导师
2. **学生资格**: 只有身份为学生(identity=3)的用户才能作为学生
3. **一对一限制**: 一个学生同时只能有一个活跃状态的导师
4. **一对多关系**: 一个导师可以同时指导多个学生
5. **状态管理**: 更换导师时，系统自动结束原有关系并创建新关系
6. **关系追踪**: 保留历史师生关系记录，支持关系变更历史查询
7. **权限控制**: 只有管理员或相关导师可以管理师生关系

### 数据一致性规则
1. **级联更新**: 更新用户信息时，相关成果的作者信息保持一致
2. **引用完整性**: 外键关系确保数据完整性
3. **事务保证**: 复杂操作使用事务确保数据一致性
4. **乐观锁**: 使用 update_time 实现乐观锁机制
5. **师生关系完整性**: 师生关系表的 teacher_id 和 student_id 必须对应有效的用户记录
